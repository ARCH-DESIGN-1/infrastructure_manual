STEP 3.2
Vault Details Keys Create Key

Generate a master encryption symmetric key inside the HSM vault boundary.

Key Name
sod-encryption-key
Algorithm & Length
AES / 32 Bytes
STEP 3.3
Vault Details Secrets Create Secret

Configure plaintext database passwords and target HCM integration credentials secrets.

Secret 1: DB Administrator Password
Name: sod-db-password | Encryption Key: sod-encryption-key | Template: Plaintext
Secret 2: HCM Integration Credentials Payload
Name: sod-hcm-credentials | Encryption Key: sod-encryption-key | Template: Plaintext
Why We Do It

Centralizing passwords inside an HSM-backed vault removes plaintext passwords from codebases and function environment variables, providing zero-trust security boundaries.
