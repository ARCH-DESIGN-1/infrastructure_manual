Networking — VCN, Gateways & Subnets

Establishes the core network boundary, regional gateways, routing rules, and isolated subnets before any other resource is provisioned.

ⓘ Prerequisites

You must have an active OCI Tenancy and a Compartment created. All resources on this page are provisioned inside that compartment. Complete Step 1 before Step 2 — subnets depend on gateways and route tables existing first.

Step 1 — Create VCN & Attach Gateways

Provision the Virtual Cloud Network container, then attach all three gateway types: Internet Gateway (public ingress), NAT Gateway (private outbound), and Service Gateway (internal OCI backbone access).

STEP 1.1
Networking Virtual Cloud Networks Create VCN

Navigate to Networking → Virtual Cloud Networks and click Create VCN. Enter the values below and click Create.

VCN Name
sod-network-vcn
IPv4 CIDR Block
10.0.0.0/16
DNS Label
sodvcn
STEP 1.2
sod-network-vcn Internet Gateways Create Internet Gateway

Open the newly created VCN. Under Resources → Internet Gateways, click Create Internet Gateway.

Gateway Name
sod-internet-gateway
Status
Enabled
STEP 1.3
sod-network-vcn NAT Gateways Create NAT Gateway

Under Resources → NAT Gateways, click Create NAT Gateway. Allows private subnet functions to call external APIs without exposing internal IPs.

Gateway Name
sod-nat-gateway
Public IP Type
Ephemeral Public IP
STEP 1.4
sod-network-vcn Service Gateways Create Service Gateway

Under Resources → Service Gateways, click Create Service Gateway. Routes database and storage traffic over Oracle's private backbone — no public internet transit.

Gateway Name
sod-service-gateway
Services
All Services in Oracle Services Network

Step 2 — Create Route Tables & Subnets

Define routing rules for each network zone, then create the public and private subnets and bind them to their respective route tables.

STEP 2.1
sod-network-vcn Route Tables Create Public Route Table

Under Resources → Route Tables, click Create Route Table. Add one route rule directing all internet traffic to the Internet Gateway.

Route Table Name
sod-public-route-table
Route Rule — Destination
0.0.0.0/0
Route Rule — Target
sod-internet-gateway
STEP 2.2
sod-network-vcn Route Tables Create Private Route Table

Create a second route table for the private subnet. Add two route rules — one for outbound NAT and one for internal Oracle service access.

Route Table Name
sod-private-route-table
Rule 1 — Destination: 0.0.0.0/0  →  Target Type
NAT Gateway → sod-nat-gateway
Rule 2 — Destination: All Oracle Services  →  Target Type
Service Gateway → sod-service-gateway
STEP 2.3
sod-network-vcn Subnets Create Public Subnet

Under Resources → Subnets, create the public subnet that will host the API Gateway endpoint.

Subnet Name
sod-public-subnet
CIDR Block
10.0.1.0/24
Subnet Access
Public Subnet
Route Table
sod-public-route-table
DNS Label
sodpub
STEP 2.4
sod-network-vcn Subnets Create Private Subnet

Create a second subnet. Select Private Subnet and check "Prohibit Public IP Addresses". This hosts Functions and the ATP Database — no public internet access.

Subnet Name
sod-private-subnet
CIDR Block
10.0.2.0/24
Subnet Access
Private Subnet (Prohibit Public IPs)
Route Table
sod-private-route-table
DNS Label
sodpriv
✓ Checkpoint — Networking Complete

At this point you should have: 1 VCN, 3 Gateways (Internet, NAT, Service), 2 Route Tables (Public + Private), and 2 Subnets (Public + Private). Proceed to Step 3 on the next page.
