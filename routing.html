Phase 6

API Gateway Routing Setup

Configures routing deployments to map public REST endpoints to private backend serverless functions.

Gateway API Endpoint Deployments

Configure a single deployment mapped in the public subnet with a prefix `/sod/v1` to host the endpoints:

Resource Endpoint HTTP Method Backend Service Target Function Design Rationale & Explanation
`/ingest` `POST` `sod-api-receiver` Ingests raw CSV payload data from enterprise clients. Validates sizes before loading to Object Storage to prevent storage exhaustion attacks.
`/connectors/run` `POST` `sod-manual-trigger` Endpoint for ad-hoc user commands to trigger specific third-party HCM system synchronization tasks immediately.
`/runs/{runId}/status` `GET` `sod-run-status` Allows clients to poll the active execution pipeline state, verifying file ingestion outcomes and retrieval rates.
`/connectors` `GET` `sod-list-connectors` Lists system-configured enterprise database API integrations (Workday, SAP SuccessFactors) available for scheduling.

Why We Do It

The API Gateway exposes public-facing endpoints to external clients while routing requests securely to functions in the private subnet. It abstracts the underlying function endpoints and enforces rate-limiting to protect the system from denial-of-service (DoS) attacks.
