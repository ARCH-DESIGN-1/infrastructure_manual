API Gateway — Public Endpoint & Route Deployment

Deploys a public API Gateway into the public subnet, then maps four REST endpoint routes to their corresponding private serverless backend functions.

ⓘ Prerequisites

All 7 functions from Step 7 must already be deployed in sod-app before configuring routes. The gateway routes reference function names directly.

API Route Mapping Reference

All routes share the base path prefix /sod/v1. Final URL format: https://<gateway-hostname>/sod/v1/<endpoint>

Route Path Method Backend Function Purpose
/ingest POST sod-api-receiver Receives audit file uploads from enterprise clients
/connectors/run POST sod-manual-trigger Triggers an on-demand HCM connector sync job
/runs/{runId}/status GET sod-run-status Returns current pipeline status for a given run
/connectors GET sod-list-connectors Lists available HCM integration connectors

Step 8 — Create Gateway, Deployment & Routes

STEP 8.1
Developer Services API Management Create API Gateway

Navigate to Developer Services → API Management → Gateways and click Create Gateway. Place it in the public subnet so it can receive internet traffic.

Gateway Name
sod-api-gateway
Type
Public
VCN
sod-network-vcn
Subnet
sod-public-subnet
STEP 8.2
sod-api-gateway Deployments Create Deployment

Open sod-api-gateway → Deployments and click Create Deployment. Set the path prefix — all routes will be relative to this base path.

Deployment Name
sod-api-v1-routes
Path Prefix
/sod/v1
STEP 8.3
Deployment Wizard Routes Tab Add Route 1 — POST /ingest

In the deployment wizard, click Add Route and configure the first endpoint. Backend type should be set to Oracle Functions.

Path
/ingest
Methods
POST
Backend Type
Oracle Functions
Function Application
sod-app
Function Name
sod-api-receiver
STEP 8.4
Deployment Wizard Routes Tab Add Remaining 3 Routes

Click Add Route three more times to add the remaining endpoints. Use Oracle Functions as backend type and sod-app as the application for all three.

Route 2
POST /connectors/run → sod-manual-trigger
Route 3
GET /runs/{runId}/status → sod-run-status
Route 4
GET /connectors → sod-list-connectors
✓ Checkpoint — API Gateway Complete

You should have 1 API Gateway (sod-api-gateway), 1 Deployment (sod-api-v1-routes with prefix /sod/v1), and 4 Routes wired to their backend functions. Note down the gateway hostname — needed to test the API. Proceed to Step 9.
