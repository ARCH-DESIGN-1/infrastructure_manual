OCI Events Rules & Alarms Setup

Configures automated event-driven hooks and defines telemetry metric alarms linked to Notification topics.

Monitoring Telemetry Alarms

Configure these 6 specific alarms inside OCI Monitoring, routing warning and critical triggers to topic `sod-alert-topic`:

Alarm Display Name Telemetry Namespace MQL Query Rule Expression Severity Design Rationale & Explanation
SOD-DLQ-Depth-CRITICAL `oci_queue` `DeadLetterQueueSize[1m]{queueId = ""}.max() > 0` `CRITICAL` Triggers if ingestion messages fail repeatedly and land in the DLQ, indicating faulty CSV file formats or column parsing exceptions.
SOD-Processor-Errors `oci_functions` `Errors[1m]{resourceId = ""}.sum() > 2` `WARNING` Alerts when the database worker loader container encounters code bugs or db driver failures.
SOD-Queue-Backlog `oci_queue` `QueueSize[5m]{queueId = ""}.mean() > 50` `WARNING` Detects queue backlog. Suggests that files are arriving faster than functions can parse them, requiring scaling tweaks.
SOD-ADB-CPU-High `oci_autonomous_database` `CpuUtilization[1m]{resourceId = ""}.mean() > 80` `WARNING` Monitors heavy query loops or flattening operations that consume database cycles, ensuring ATP auto-scaling activates.
SOD-ADB-Storage-Critical `oci_autonomous_database` `StorageUtilization[1m]{resourceId = ""}.max() > 85` `CRITICAL` Triggers if database table tablespaces fill up, indicating the need to run archiving/truncating operations.
SOD-Connector-DLQ-Depth `oci_queue` `DeadLetterQueueSize[1m]{queueId = ""}.max() > 0` `CRITICAL` Fires if third-party integration runs fail repeatedly, signaling API access key expires or network connection cuts.
