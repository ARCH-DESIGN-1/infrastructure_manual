Phase 3

OCI Messaging Queues Setup

Configures the decoupling queues and Dead Letter Queue (DLQ) pathways for file-ingestion retry tolerance.

Queue Configuration Parameters

The table below summarizes the exact values to input during provisioning:

Queue Name Visibility Lock (Timeout) Max Retries before DLQ Associated DLQ Name Design Rationale & Explanation
sod-ingestion-dlq `30` seconds N/A N/A (Target DLQ) Quarantine storage for malformed audit uploads. Low visibility timeout permits quick manual inspect/reprocessing routines without message lock delays.
sod-ingestion-queue `300` seconds (5 minutes) 3 attempts `sod-ingestion-dlq` Buffers file metadata from storage events. The 300s visibility lock allows database load routines ample time to finish ingestion before message recycling.
sod-connector-dlq `30` seconds N/A N/A (Target DLQ) Holds failed integration sync runs (e.g. Workday network drops), enabling system admins to review payloads before reprocessing.
sod-connector-queue `600` seconds (10 minutes) 3 attempts `sod-connector-dlq` Buffers external sync commands. 600s visibility prevents duplicate worker checkouts on slow multi-megabyte API downloads.

Step 5 — Configure Queues & DLQ Pairs

Decouple ingestion processes by deploying transaction queues and dead-letter targets to catch ingestion errors.

STEP 5.1
Developer Services Queues Create DLQs (First)

Provision target Dead Letter Queues (DLQs) to store unprocessable messages safely.

Queue 1: Ingestion Dead Letter Queue
Name: sod-ingestion-dlq | Visibility: 30 seconds | Message Retention: 86400 seconds
Queue 2: Connector Dead Letter Queue
Name: sod-connector-dlq | Visibility: 30 seconds | Message Retention: 86400 seconds
STEP 5.2
Developer Services Queues Create Primary Queues

Configure primary ingestion and external connector messaging queues, linking them to their corresponding DLQs.

Primary Ingestion Queue Configuration
Name: sod-ingestion-queue | Visibility: 300 seconds | Retention: 86400 seconds | DLQ: sod-ingestion-dlq | Max Delivery Attempts: 3
Primary Connector Queue Configuration
Name: sod-connector-queue | Visibility: 600 seconds | Retention: 86400 seconds | DLQ: sod-connector-dlq | Max Delivery Attempts: 3
Why We Do It

Queues decouple ingestion triggers from database writes. If processing is interrupted or files are malformed, the visibility lock expires and OCI Queue automatically retries it up to 3 times before routing the failing file to the DLQ, ensuring the main pipeline continues processing subsequent files.
