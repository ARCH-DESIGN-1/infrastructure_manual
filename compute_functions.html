OCIR & Functions — Registry, Application & Serverless Deployment

Creates a private container image registry, builds and pushes Docker images for each function, creates the Functions Application, then deploys all 7 serverless function definitions.

ⓘ Prerequisites

You must have Docker installed locally and access to the SoD platform source code repository. Have your OCI tenancy namespace ready — find it at Profile → Tenancy → Object Storage Namespace.

Function Deployment Parameters

Reference table — click any row to view the full source code for that function:

Function Name Memory (MB) Timeout (s) Purpose
sod-api-receiver 256 60 Receives inbound audit file uploads via the public API endpoint
sod-enqueuer 128 60 Triggered by bucket upload events — enqueues file metadata to ingestion queue
sod-processor 256 300 Dequeues messages and loads raw CSV/JSON data into ATP staging tables
sod-manual-trigger 256 30 Allows on-demand trigger of specific HCM connector sync jobs via API
sod-connector-processor 256 600 Polls HCM APIs (Workday, Oracle HCM), downloads data, deposits to bucket
sod-run-status 256 30 Returns current run pipeline state and processing stage for a given run ID
sod-list-connectors 256 30 Lists all active HCM connector configurations stored in the database

Step 7 — Deploy Container Registry & Functions

STEP 7.1
Developer Services Container Registry Create Repository

Navigate to Developer Services → Container Registry and click Create Repository. Set access to Private — this is where all Docker function images are stored.

Repository Name
sod-repo
Access
Private
STEP 7.2
Local Terminal Docker CLI Login to OCIR & Build + Push Images

From your local terminal, log in to OCIR, then build and push each function image. Repeat the build and push commands for all 7 functions.

1. Docker login command
<region-key>.ocir.io
Username format
<tenancy-namespace>/<oci-username>
2. Image tag format (repeat per function)
<region-key>.ocir.io/<tenancy-namespace>/sod-repo/<fn-name>:1.0.0
STEP 7.3
Developer Services Functions Create Application

Navigate to Developer Services → Functions → Applications and click Create Application. Bind it to the private subnet so all functions can access the ATP database internally.

Application Name
sod-app
VCN
sod-network-vcn
Subnets
sod-private-subnet
STEP 7.4
sod-app Functions Create Function (×7)

Inside sod-app, click Create Function. Repeat for all 7 functions, using the Docker image tag from Step 7.2 and the memory/timeout values from the reference table above.

Image format for each function
ap-mumbai-1.ocir.io/<namespace>/sod-repo/<fn-name>:1.0.0
Functions to create (in order)
sod-api-receiver | sod-enqueuer | sod-processor | sod-manual-trigger | sod-connector-processor | sod-run-status | sod-list-connectors
✓ Checkpoint — Functions Complete

You should have 1 OCIR Repository (sod-repo with 7 images), 1 Functions Application (sod-app), and 7 Functions deployed inside it. Proceed to Step 8 to configure the API Gateway.
