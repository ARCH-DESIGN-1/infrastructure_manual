Phase 5

Private Registry & Serverless Functions Setup

Configures container registry parameters and deploys the private serverless execution application.

Serverless Deployment Parameters

Identify individual function properties and memory profiles before push execution:

Function Name Memory (MB) Timeout (s) Container Image Target Path Design Rationale & Explanation
sod-api-receiver `256` `60` `ap-mumbai-1.ocir.io//sod-repo/sod-api-receiver:1.0.0` Handles public API ingestion. Validates JWT security headers and schema bounds, returns a run ID immediately.
sod-enqueuer `128` `60` `ap-mumbai-1.ocir.io//sod-repo/sod-enqueuer:1.0.0` Enqueues storage event file metadata onto the ingestion queue. Requires low footprint and executes quickly.
sod-processor `256` `300` `ap-mumbai-1.ocir.io//sod-repo/sod-processor:1.0.0` Parses raw CSV/JSON logs and populates staging database rows. 300s window allows parsing of multi-megabyte audit files.
sod-manual-trigger `256` `30` `ap-mumbai-1.ocir.io//sod-repo/sod-manual-trigger:1.0.0` Enables on-demand ad-hoc triggers of specific HCM enterprise connectors via API request paths.
sod-connector-processor `256` `600` `ap-mumbai-1.ocir.io//sod-repo/sod-connector-processor:1.0.0` Polls external HCM targets (e.g. Workday, Oracle HCM REST) and dumps logs to landing buckets. Long timeout handles large remote datasets.
sod-run-status `256` `30` `ap-mumbai-1.ocir.io//sod-repo/sod-run-status:1.0.0` Serves GET status updates. Reads active run metadata indexes to track processing stages and violations.
sod-list-connectors `256` `30` `ap-mumbai-1.ocir.io//sod-repo/sod-list-connectors:1.0.0` Lists active HCM target connectors defined inside the SQL database lookup metadata schemas.

Step 7 — Deploy Containerized Functions

Configure private docker registry storage and deploy serverless functions matching regional limits.

STEP 7.1
Developer Services Container Registry Create Repository

Provision a secure repository folder inside the Oracle Cloud Infrastructure Registry (OCIR).

Repository Name
sod-repo
Access Type
Private
STEP 7.2
Local Terminal Docker CLI Build & Push

Log in to the regional OCIR endpoint and push tag-matched docker worker images.

Docker Login Registry command
docker login <region-key>.ocir.io
OCI Destination Registry Tag format
<region-key>.ocir.io/<tenancy-namespace>/sod-repo/<fn-name>:1.0.0
STEP 7.3
Developer Services Functions Create Application

Establish the core serverless container application boundary linked to the private database subnet.

Application Name
sod-app
Network VCN Bind
sod-network-vcn
Subnet Bind
sod-private-subnet
STEP 7.4
sod-app Details Functions List Create Function

Map the individual function runtimes, matching memory profiles and timeouts exactly from the telemetry sheet.

Setup Guideline
Provision all 7 functions matching names, memory allocations, and Docker tags.
Why We Do It

Deploying OCI Functions inside the private subnet ensures they can access the database directly via internal endpoints and query OCI services securely over NAT and Service Gateways.
